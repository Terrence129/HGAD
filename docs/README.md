# Documentation Overview

This `docs/` directory contains project documentation for the HGAD thesis project. It is organized to support academic writing, progress tracking, meeting records, and versioned thesis drafts.

## Directory Structure

- `docs/literature/`
  - Contains literature notes, paper summaries, and annotated references used during thesis research.
- `docs/reports/`
  - Stores interim reports, progress summaries, and project review documents.
- `docs/meetings/`
  - Stores meeting minutes, supervisor discussions, and action item records.
- `docs/thesis/`
  - Contains thesis drafts, changelogs, and version-controlled draft material.

## Purpose of Each Subdirectory

- `literature`: Record academic sources, Zotero export notes, and synthesis of related work.
- `reports`: Provide periodic status updates, interim reports, and progress assessments.
- `meetings`: Capture formal meeting details, agenda items, and follow-up tasks.
- `thesis`: Maintain structured versions of the thesis draft, changelog history, and final composition artifacts.

## Naming Conventions

- Use ISO date formatting for meeting notes and time-sensitive records, e.g. `2026-05-05_meeting_minutes.md`.
- Thesis drafts should include version metadata, e.g. `thesis_draft_v1.0.md`.
- Changelogs should follow a versioned pattern, e.g. `changelog_v1.0.md`.
- Interim reports should include version and report type, e.g. `interim_report_v1.0.md`.

## Backup and Version Control

- Primary version control is GitHub. Commit documentation updates alongside code changes whenever possible.
- Use descriptive commit messages for documentation changes, such as `docs: add thesis draft template`.
- Backup critical documents to Baidu Drive regularly. Keep at least one synchronized copy for each major milestone.
- Preserve historical versions of drafts and reports in the `docs/thesis/changelogs/` directory and maintain a clean, traceable revision history.
